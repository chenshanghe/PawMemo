1. 产品概述

1.1 产品定位与愿景

顽童日记是一款面向中文用户的 AI 辅助旅行日记 Web 应用，让每个旅行者都能轻松记录零散旅途笔记，并借助 AI 一键整合成完整、流畅的游记文章。

核心理念
旅行中随手记，回来后 AI 帮你整理成文。不需要会写作，也能拥有一篇有质感的旅行故事。

1.2 目标用户

用户群体特征核心诉求
普通旅行爱好者频繁出行，不擅长写作快速记录，AI 整理成文
旅行博主有写作习惯，需要提效素材管理、AI 润色、一键分享
家庭出行记录者带家人旅行，想留存回忆图文混排、精美排版、私密保存
背包客长途旅行，随记多离线使用、批量整理、足迹地图

1.3 核心价值主张

  • 低门槛记录：随记短写，无需构思全文结构
  • AI 增值：多篇随记 → AI 自动合成完整游记，严格忠实原文，不编造内容
  • 内容所有权：私密/公开权限精细控制，分享链接带 token 保护
  • 社交发现：广场浏览他人游记，点赞、评论、关注
  • 跨设备：PWA + 离线缓存，已读内容断网可访问

2. 功能模块详述

2.1 随记（旅行笔记）

随记是最轻量的记录单元，对应路由 /entries/new，类型标识 entryType: "note"。

字段类型说明
标题文本必填，简短描述
正文富文本支持换行，内容主体
目的地文本城市/景点名，用于地图和筛选
开始/结束日期日期支持单日或日期范围
心情枚举开心/平静/感动/疲惫/兴奋/思念
标签多选自定义标签，支持筛选
图片多图支持 HEIC 转换，自动上传至对象存储
隐私枚举私密 / 公开 / 好友可见

2.2 游记（长篇旅行文章）

游记对应类型 entryType: "narrative"，通常由 AI 合成或手动创建，展示页支持：

  • 图片列数选择（1列/2列/3列），排版偏好持久化
  • 打印友好视图（/entries/:id/print）
  • 字体为 Noto Serif SC，保持日记质感

2.3 AI 合成游记

路由：/entries/compose，后端：POST /api/ai/compose（DeepSeek SSE 流式输出）。

Prompt 五大原则（2026.06 最新版）
1. 严格按时间先后顺序组织全文
2. 不得编造任何人物、地点、情节、对话或心理活动
3. 可添加连接词/过渡句，但不得增添原文未提及的内容
4. 时间标记自然融入叙事，避免罗列式写法
5. 综合所有随记内容拟定标题（格式 【标题】xxx，前端自动提取预填）
功能说明
选择来源选 2–10 篇随记，按时间排序
写作风格可输入自定义风格要求；支持保存风格预设
配图选择按来源随记分组选图，嵌入对应段落
标题自动提取AI 输出 【标题】xxx，前端解析并预填标题输入框
分段标记[===] 分隔符对应每篇随记，图片插入对应段后

2.4 AI 旅行规划

路由：/plan，后端：POST /api/ai/plan。

  • 输入：出发地、目的地、日期范围、人数、出行方式、预算、特殊需求、风格偏好
  • 输出：流式生成详细行程计划
  • 支持保存计划（/plan/list），后续重新规划时字段变更高亮显示
  • 重规划横幅：Amber 色提示正在修改已保存计划，可手动关闭

2.5 足迹地图

路由：/map，使用 Leaflet.js。

  • 自动聚合用户所有日记的目的地，显示为地图标记
  • 点击标记查看该地的日记列表
  • Tailwind v4 下需特殊 CSS 修复（.leaflet-container img.leaflet-tile 防止图片坍缩）

2.6 社交与广场

功能路由说明
广场/square浏览所有公开游记，按时间倒序
动态/feed关注用户的最新动态
点赞详情页离线时给出 Amber 提示，不静默失败
评论详情页离线时隐藏输入框，显示提示文案
关注用户主页/users/:userId

2.7 分享与公开访问

  • 每篇日记可生成唯一分享 token：POST /api/entries/:id/share
  • 公开链接：/share/:token，无需登录可访问
  • ACL 规则：同一图片在公开和私密日记间共用时，取最宽松权限（most-permissive wins）
  • 分享 token 防重复：withToken() 函数幂等，storage.ts 处理数组型参数

2.8 相册管理

路由：/photos。

  • 按日记分组展示所有上传图片
  • 支持 HEIC 格式（前端 heic2any 转换）
  • 图片存储：Replit Object Storage，公开/私密 ACL 精细控制

2.9 离线支持 / PWA

  • Service Worker（sw.js）独立图片缓存（最多 500 张，FIFO 淘汰）
  • API 数据：stale-while-revalidate 策略
  • 已访问图片：cache-first，离线时未缓存图片返回透明占位图
  • 顶部"离线模式"横幅（amber 色）
  • PWA 安装提示 + 版本更新提示（显示构建日期 YYYY.MM.DD）

5. 用户旅程

5.1 新用户首次使用

  1. 访问 urchins.life → 落地页（未登录状态）
  2. 点击注册 → Clerk 注册流程（邮箱/第三方）
  3. 注册完成 → 自动同步用户 profile 到数据库（POST /api/me/profile）
  4. 跳转 Dashboard → 首次引导，新建第一篇随记
  5. 随记创建 → 出现在 Dashboard「最近日记」列表

5.2 AI 合成游记完整流程

  1. 在 /entries 点击「合成随记成游记」→ 进入选择模式
  2. 勾选 2–10 篇随记 → 点击「前往合成」→ 携带 ?ids=1,2,3 跳转到 /entries/compose
  3. （可选）填写写作风格要求 / 选择已保存风格预设
  4. 点击「开始合成」→ SSE 流式输出开始
  5. AI 第一行输出 【标题】xxx → 前端自动提取并预填标题输入框
  6. 正文流式渲染完毕，[===] 分段自动对应各随记
  7. （可选）选择配图，插入对应段落
  8. 点击「保存游记」→ 新游记出现在 /entries 列表

5.3 分享流程

  1. 进入日记详情页 → 点击「分享」图标
  2. 打开分享面板 → 点击「生成链接」
  3. 系统调用 POST /api/entries/:id/share 生成 shareToken
  4. 复制链接 urchins.life/share/:token
  5. 接收方无需登录即可访问，显示完整日记内容和图片

5.4 升级付费流程

  1. 触发配额限制 → 弹出升级提示 / 主动前往 /pricing
  2. 选择套餐（Pro / Plus）和计费周期（月付 / 年付）
  3. 点击「立即升级」→ 弹出支付弹窗
  4. 展示微信支付二维码（后端创建迅虎订单，返回 QR URL）
  5. 扫码完成支付 → 前端每 3 秒轮询订单状态
  6. 轮询到 status: "paid" → 显示成功，自动刷新订阅状态
  7. 迅虎同时发送 Webhook → 后端双重 fulfill 保障
