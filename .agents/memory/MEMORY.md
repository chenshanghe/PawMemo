- [Share page image double shareToken](share-page-image-double-token.md) — backend + frontend both appended shareToken → array query param → 403; withToken() must be idempotent, storage.ts must handle array.
- [Tailwind v4 breaks Leaflet tiles](tailwind-leaflet-fix.md) — Tailwind's base `img` styles (max-width:100%, height:auto, display:block) collapse Leaflet tile images to 0 size; must override with `.leaflet-container img.leaflet-tile` CSS in index.css.
- [react-simple-maps JSON import](react-simple-maps-json.md) — must import TopoJSON as module, not string URL; Vite cold restart leaves browser on old bundle, HMR alone insufficient — restart workflow to force full reload.
- [BASE_URL required for all fetch calls](base-url-fetch.md) — app is mounted at /travel-diary/ so bare /api/... paths silently 404; every fetch must use `${BASE}/api/...` where BASE = import.meta.env.BASE_URL.replace(/\/$/, "").
- [Travel Diary Production Build](travel-diary-build.md) — deployment serves pre-built static dist, NOT source; must run `PORT=18469 BASE_PATH=/ pnpm --filter @workspace/travel-diary run build` before every Publish.
- [Drizzle + Express 5 type gotchas](drizzle-express5-types.md) — three recurring TS issues: (1) zod.coerce.date() → Date but Drizzle date() expects string; (2) req.params.xxx is string|string[] in Express 5 types — need `as string`; (3) inline object literals trigger false-positive excess property checks on complex Drizzle overloads — extract to a named variable first.
- [xunhupay signing algorithm](xunhupay-signing.md) — sign = md5(sortedParams + APPKEY) with NO "&key=" separator; url_qrcode is already a QR image (display directly, don't re-encode).
- [Vite proxy required for dev API routing](vite-proxy-dev.md) — browser requests all hit Vite (port 18469); without proxy `/api` never reaches Express (port 8080); must add `server.proxy: { "/api": "http://localhost:8080" }` in vite.config.ts.
- [Canvas image assets must go via .canvas/assets/](canvas-image-upload.md) — canvas rejects arbitrary https URLs for images; copy files to .canvas/assets/ first, then serve via https://<DOMAIN>:5904/filename (no query params, no __mockup base path).
